Доработай уже существующий проект свадебного сайта-приглашения. Не создавай всё заново с нуля — именно внеси изменения в текущий проект, исправь недочёты и улучши архитектуру, UI и функциональность.

Главная цель доработки

Нужно исправить проблемы с копированием ссылок, добавить удобную отправку приглашений через Telegram, сделать корректную разметку для соцсетей и сильно улучшить игру. Игра не обязана быть 3D — можно сделать 2D, если так она будет качественнее, стабильнее и удобнее. Главное, чтобы она реально хорошо игралась на мобильных, планшетах и ПК.

1. Исправить копирование ссылки

Сейчас ссылка копируется некорректно или не полностью. Исправь это.

Что нужно:
	•	Кнопка “Копировать ссылку” должна копировать полный абсолютный URL, а не относительный путь.
	•	Должна копироваться ссылка целиком, например:
https://my-app.replit.app/invite/anna-ivanova-ab12cd
	•	Нельзя копировать сокращённый путь вида:
/invite/anna-ivanova-ab12cd
	•	Проверь, чтобы ссылка бралась из корректного base URL.
	•	Используй:
	•	window.location.origin
	•	либо настройку APP_BASE_URL из env
	•	Сделай fallback, если navigator.clipboard.writeText() не работает:
	•	через скрытый input
	•	select
	•	document.execCommand('copy')
	•	После копирования должно показываться понятное уведомление:
	•	“Ссылка скопирована”
	•	либо сообщение об ошибке

Дополнительно:
	•	В админке рядом с каждой персональной ссылкой покажи:
	•	саму полную ссылку
	•	кнопку копирования
	•	кнопку открытия ссылки в новой вкладке

⸻

2. Добавить быструю отправку ссылки в Telegram

Нужна удобная отправка персонального приглашения в Telegram.

Что нужно:
	•	В админке рядом с каждой записью гостя добавь кнопку:
	•	“Отправить в Telegram”
	•	При нажатии должна открываться готовая ссылка шаринга:
https://t.me/share/url?url=FULL_URL&text=TEXT
	•	В url должна подставляться полная персональная ссылка гостя
	•	В text должен подставляться красивый текст, например:
“Ваше персональное приглашение на нашу свадьбу 💍”
	•	Все параметры должны быть корректно URL-encoded
	•	Ссылка Telegram должна работать на мобильных и десктопе

Дополнительно:

Добавь ещё кнопку:
	•	“Скопировать текст + ссылку”

Чтобы копировалось готовое сообщение типа:
“Ваше персональное приглашение на нашу свадьбу 💍
https://my-app.replit.app/invite/anna-ivanova-ab12cd”

3. Сделать разметку для соцсетей

Добавь полноценную SEO и social sharing разметку для корректного предпросмотра ссылки в соцсетях и мессенджерах.

Обязательно:

Добавь в страницу приглашения:
	•	title
	•	meta description
	•	Open Graph tags
	•	Twitter Card tags

Нужные теги:
	•	og:title
	•	og:description
	•	og:image
	•	og:url
	•	og:type
	•	twitter:card
	•	twitter:title
	•	twitter:description
	•	twitter:image

Требования:
	•	При открытии ссылки в Telegram, WhatsApp, Facebook и других соцсетях должен быть красивый предпросмотр
	•	Должна быть обложка приглашения
	•	Должен быть заголовок
	•	Должно быть короткое описание
	•	Должен использоваться абсолютный URL
	•	Если проект SSR не использует, продумай способ серверной выдачи meta для invite-страниц или сделай рабочее решение для Replit

Дополнительно:

Добавь настройки в админке или в settings:
	•	заголовок предпросмотра
	•	описание предпросмотра
	•	картинка предпросмотра
	•	базовый URL проекта

4. Сильно доработать игру

Текущая игра сделана плохо или слишком сыро. Её нужно полностью улучшить. Можно отказаться от 3D, если 3D мешает качеству. Лучше сделать отличную 2D-игру, чем слабую 3D-демку.

Главное требование к игре

Игра должна быть:
	•	лёгкой
	•	красивой
	•	понятной
	•	стабильной
	•	плавной
	•	адаптивной
	•	удобной на телефоне, планшете и ПК
	•	не раздражающей
	•	тематически связанной со свадьбой

⸻

5. Новый концепт игры

Сделай качественную свадебную мини-игру перед открытием приглашения.

Предлагаемая идея:

Игрок помогает соединить жениха и невесту или собрать свадебные символы любви, чтобы открыть приглашение.

Допустимые механики:

Выбери одну лучшую и реализуй качественно:

Вариант A
Игрок управляет сердцем, которое собирает:
	•	кольца
	•	цветы
	•	сердца
	•	лепестки
	•	звёзды

Вариант B
Игрок помогает жениху дойти до невесты, избегая препятствий и собирая бонусы любви

Вариант C
Нужно собрать несколько свадебных предметов, после чего открывается приглашение

⸻

6. Требования к новой игре

По геймплею:
	•	быстрый старт
	•	понятное управление с первого экрана
	•	короткая длительность: 20–60 секунд
	•	1 простая цель
	•	несколько попыток
	•	плавная анимация
	•	нормальная отзывчивость управления
	•	без перегруженности

По адаптации:
	•	мобильные устройства
	•	планшеты
	•	ПК
	•	корректный scaling canvas
	•	touch controls для телефона
	•	клавиатура для ПК
	•	кнопки управления на экране для мобильных

По UX:
	•	экран приветствия перед игрой
	•	краткая инструкция
	•	индикатор прогресса
	•	состояние победы
	•	состояние проигрыша
	•	кнопка “Играть снова”
	•	кнопка “Пропустить” можно не делать, если по логике нужно обязательно пройти игру

По стилю:
	•	свадебная тема
	•	романтичные пастельные цвета
	•	мягкие анимации
	•	жених и невеста должны присутствовать визуально
	•	можно в виде иллюстраций, 2D-персонажей, силуэтов или стилизованных фигур
	•	добавить частицы, лепестки, свечение, сердечки

⸻

7. Техническая реализация игры

Используй тот подход, который даст лучший результат в Replit:
	•	Phaser.js
или
	•	Canvas 2D
или
	•	React game loop

Приоритет:

Не “эффектно”, а “стабильно и играбельно”.

Реализуй:
	•	отдельный модуль игры
	•	компонент игры
	•	экран загрузки игры
	•	сохранение факта прохождения игры
	•	после победы — анимация открытия приглашения
	•	запись в БД:
	•	gameCompleted = true
	•	защита от багов повторного открытия
	•	при повторном входе можно либо показывать игру снова, либо сразу открывать приглашение, если она уже пройдена
